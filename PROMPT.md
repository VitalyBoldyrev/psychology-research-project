# TELEGRAM-БОТ ДЛЯ ПСИХОЛОГИЧЕСКОГО ИССЛЕДОВАНИЯ ЛОГИЧЕСКОГО МЫШЛЕНИЯ

## КОНТЕКСТ ПРОЕКТА
Создай Telegram-бота на Python для проведения психологического исследования логического мышления среди студентов. Бот собирает демографические данные, проводит тестирование с ~30 задачами на логику, сохраняет все данные в Google Sheets и выдает логин для прохождения второй части исследования на внешнем сайте.

## ТЕХНИЧЕСКИЙ СТЕК
- Python 3.10+
- aiogram 3.x (библиотека для Telegram Bot API)
- Google Sheets API (gspread + google-auth)
- Асинхронное программирование (asyncio)
- Планировщик задач: APScheduler (для таймеров и напоминаний)

## АРХИТЕКТУРА ДАННЫХ

### Google Sheets - структура таблицы
Все данные хранятся в одной Google Sheets таблице со следующими колонками:

| unique_id | telegram_login | telegram_phone | name | age | education | question_1 | question_2 | ... | question_30 | website_completed | registration_date | test_start_time | test_end_time |

**Описание полей:**
- `unique_id` - уникальный идентификатор в формате km001, km002, km003... (генерируется автоматически)
- `telegram_login` - username пользователя в Telegram (@username) или пусто
- `telegram_phone` - номер телефона если нет username (запрашивается через встроенную кнопку Telegram)
- `name` - имя студента
- `age` - возраст
- `education` - уровень образования (одно из: Среднее общее, Среднее специальное, Неполное высшее, Высшее)
- `question_1` ... `question_30` - ответы на вопросы по логическому мышлению
- `website_completed` - флаг прохождения тестирования на сайте (YES/NO, изначально NO)
- `registration_date` - дата и время первого запуска бота
- `test_start_time` - время начала тестирования в боте
- `test_end_time` - время завершения тестирования в боте

### Логика работы с Google Sheets

**При первом запуске /start:**
1. Проверить, есть ли уже запись с данным telegram_id
2. Если НЕТ - создать новую строку:
   - Найти максимальный unique_id (например km015)
   - Присвоить следующий (km016)
   - Записать telegram_login (если есть username: @username)
   - Если username отсутствует - запросить номер телефона через кнопку "Поделиться номером телефона"
   - Записать registration_date (текущее время)
3. Если ДА - продолжить с существующей записью (позволяет возобновить прохождение)

**Важно:** Google Sheets используется как единственное хранилище данных, локальная БД НЕ нужна.

## ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

### 1. РЕГИСТРАЦИЯ И ИДЕНТИФИКАЦИЯ

**Сценарий первого запуска /start:**

1. **Приветственное сообщение**
2. **Получение telegram username или телефона:**
   - Если у пользователя есть @username - сохранить в telegram_login
   - Если НЕТ - показать кнопку "Поделиться номером телефона" (KeyboardButton с request_contact=True)
   - Сохранить телефон в формате +7XXXXXXXXXX в telegram_phone
3. **Генерация unique_id:**
   - Прочитать все существующие unique_id из таблицы
   - Найти максимальное число (например, km015 → 15)
   - Присвоить следующее: km016
   - Записать в таблицу новую строку
4. **Последовательный сбор данных:**
   - Запрос имени (текстовый ввод)
   - Запрос возраста (числовой ввод с валидацией 14-100)
   - Выбор образования через inline-кнопки:
     * Среднее общее
     * Среднее специальное
     * Неполное высшее
     * Высшее
5. **Обновление записи в Google Sheets** после каждого введенного поля

**Требования:**
- Валидация каждого поля перед переходом к следующему
- Возможность вернуться на шаг назад (кнопка "◀️ Назад")
- Автоматическое сохранение прогресса в Google Sheets

### 2. СИСТЕМА ТЕСТИРОВАНИЯ

**Хранение вопросов:**
Вопросы хранятся в **отдельном листе** той же Google Sheets таблицы с названием "Questions"

Структура листа Questions:
| question_id | question_text | question_type | options | is_active | order_number |

- `question_id` - уникальный ID вопроса (1, 2, 3...)
- `question_text` - текст вопроса (например: "Даны 3 шара разного цвета. Сколько пар шаров разного цвета можно составить?")
- `question_type` - тип ответа: "number" (числовой ввод), "choice" (выбор варианта), "yesno" (да/нет)
- `options` - варианты ответа для типа "choice", разделенные символом "|" (например: "2|3|4|6" или пусто для number/yesno)
- `is_active` - активен ли вопрос (TRUE/FALSE)
- `order_number` - порядковый номер отображения (1-30)

**Типы вопросов:**

1. **Числовой ввод (number):**
   - Пользователь вводит число
   - Сохраняется как есть

2. **Выбор варианта (choice):**
   - Отображаются inline-кнопки с вариантами из поля options
   - Пример: кнопки "2", "3", "4", "6"
   - Сохраняется выбранный вариант

3. **Да/Нет (yesno):**
   - Две кнопки: "Да" и "Нет"
   - Сохраняется "Да" или "Нет"

**ВАЖНО:** Пользователю НЕ сообщается тип вопроса - он должен сам понять как отвечать по формулировке задачи (это часть проверки мышления).

**Логика прохождения:**
1. При старте тестирования записать test_start_time в Google Sheets
2. Загрузить из листа "Questions" все активные вопросы (is_active=TRUE), отсортированные по order_number
3. Задавать вопросы последовательно
4. Показывать прогресс: "Вопрос 5 из 28" (только активные)
5. Сохранять ответ в соответствующую колонку question_N в Google Sheets после каждого ответа
6. **Возможность вернуться назад:**
   - Кнопка "◀️ Изменить предыдущий ответ" под каждым вопросом
   - При возврате показать предыдущий вопрос с текущим ответом
   - Разрешить изменить ответ
7. После последнего вопроса записать test_end_time → переход к выдаче логина

**Индикация:**
```
📊 Вопрос 5 из 28

Даны 3 шара разного цвета. Сколько пар шаров разного цвета можно составить?

[Для числового: пользователь вводит число]
[Для выбора: кнопки с вариантами]
[Для да/нет: кнопки Да/Нет]

[КНОПКА: ◀️ Изменить предыдущий ответ]
```

### 6. ИНТЕГРАЦИЯ С GOOGLE SHEETS

**Настройка Google Sheets API:**

1. Создать проект в Google Cloud Console
2. Включить Google Sheets API
3. Создать Service Account
4. Скачать JSON-ключ
5. Предоставить доступ к таблице для email Service Account

**Библиотеки:**
```python
import gspread
from google.oauth2.service_account import Credentials
```

**Инициализация в коде:**
```python
SCOPES = ['https://www.googleapis.com/auth/spreadsheets']
CREDENTIALS_FILE = 'credentials.json'  # JSON-ключ от Service Account
SPREADSHEET_ID = 'ваш_id_таблицы'  # из URL таблицы

creds = Credentials.from_service_account_file(CREDENTIALS_FILE, scopes=SCOPES)
client = gspread.authorize(creds)
spreadsheet = client.open_by_key(SPREADSHEET_ID)

# Рабочие листы
main_sheet = spreadsheet.worksheet('Main')  # основные данные
questions_sheet = spreadsheet.worksheet('Questions')  # вопросы
timers_sheet = spreadsheet.worksheet('Timers')  # таймеры (опционально)
```

**Функции для работы с данными:**

1. `create_new_user(telegram_id, username, phone)` - создать новую строку с уникальным km00X
2. `update_user_field(telegram_id, field_name, value)` - обновить конкретное поле
3. `get_user_by_telegram_id(telegram_id)` - получить данные пользователя
4. `save_answer(telegram_id, question_number, answer)` - сохранить ответ на вопрос
5. `get_all_questions()` - получить все активные вопросы из листа Questions
6. `add_question(text, type, options)` - добавить новый вопрос
7. `update_question(question_id, field, value)` - обновить вопрос
8. `delete_question(question_id)` - деактивировать вопрос
9. `mark_website_completed(telegram_id)` - установить website_completed = "YES"

**Оптимизация:**
- Использовать batch_update() для множественных операций
- Кэширование вопросов в памяти бота (перезагрузка раз в 5 минут)

### 7. ОБРАБОТКА ОШИБОК И UX

**Валидация данных:**
- Возраст: только числа 14-100
- Имя: не пустое, минимум 2 символа
- Телефон: автоматически из Telegram (не требует валидации)

**Сообщения об ошибках:**
```
❌ Пожалуйста, введите корректный возраст (число от 14 до 100)

❌ Имя должно содержать минимум 2 символа

❌ Пожалуйста, введите число в качестве ответа

❌ Произошла ошибка при сохранении данных. Попробуйте еще раз.
```

**Логирование:**
- Все ошибки записывать в файл `bot_errors.log`
- Формат: `[timestamp] ERROR: описание ошибки + traceback`
- При критической ошибке - уведомление администратору в личные сообщения

**Возобновление сессии:**
- Если пользователь повторно нажал /start:
  - Проверить его статус в таблице
  - Если не завершил регистрацию - продолжить с места остановки
  - Если не завершил тестирование - спросить: "Продолжить тестирование? [Да] [Начать заново]"
  - Если завершил всё - показать его unique_id и ссылку на сайт

**UX улучшения:**
- Эмодзи для визуального разделения блоков
- Прогресс-бары: "▓▓▓▓▓▓░░░░ 60%"
- Inline-кнопки вместо текстовых команд
- Приятные сообщения: "Отлично! ✨", "Супер! 🎯", "Спасибо! 🙏"
